**Step 3: Modify Step 5 in analyzeWorldwideRemote**

Replace the Step 5 block (lines 282-303) with:

```typescript
  // --- STEP 5: If location is ambiguous, check for worldwide signal ---
  if (!locationIsExactWorldwide) {
    // 5a: Trusted companies get a pass for bare "Remote" locations
    if (job.companySlug && TRUSTED_WORLDWIDE_SLUGS.has(job.companySlug)) {
      const locLower = (job.location || "").toLowerCase().trim();
      if (locLower.includes("remote")) {
        return { pass: true, reason: "pass_trusted_company" };
      }
    }
  }

  // --- STEP 6: If no location was provided, require worldwide signal in text ---
  if (!location) {
    const textToCheck = [job.title || "", job.description || ""]
      .join(" ")
      .toLowerCase();
    const hasWorldwideSignal =
      textToCheck.includes("worldwide") ||
      textToCheck.includes("work from anywhere") ||
      textToCheck.includes("globally") ||
      textToCheck.includes("any location") ||
      textToCheck.includes("no location requirement") ||
      textToCheck.includes("location independent") ||
      textToCheck.includes("open to candidates globally") ||
      textToCheck.includes("hire from anywhere") ||
      textToCheck.includes("no geographic restriction") ||
      textToCheck.includes("distributed team") ||
      textToCheck.includes("remote friendly") ||
      textToCheck.includes("across the globe");
    if (!hasWorldwideSignal) {
      return { pass: false, reason: "no_worldwide_signal" };
    }
  }
```

**Files:**
- Create: `scripts/harvest-ats.ts`

**Step 1: Build the slug harvester script**

This script takes a different approach from the previous discovery pipeline. Instead of starting from company lists, it directly enumerates ATS slugs by:

1. Fetching Google search results for `site:boards.greenhouse.io`, `site:jobs.lever.co`, `site:jobs.ashbyhq.com`
2. Extracting slugs from URLs
3. Deduping against existing companies.ts
4. Verifying each slug has active jobs via ATS API
5. Sampling jobs through the worldwide filter
6. Outputting merge-ready entries

Since Google Custom Search API requires an API key, build an alternative approach: **brute-force slug testing**. Generate slug candidates from:
- YC company directory (algolia API)
- Crunchbase-style company name lists (public datasets)
- Common English words and company name patterns
- The scripts/output/ discovered.csv results that had "no ATS" — try alternate slug patterns

The script should:
```typescript

// 1. Load company name candidates from multiple sources
// 2. Generate slug permutations per name (lowercase, hyphenated, domain-based)
// 3. Test each slug against Greenhouse, Lever, Ashby, Workable APIs
// 4. Filter out slugs already in companies.ts
// 5. Sample 3 jobs per slug through isWorldwideRemote()
// 6. Output: new companies with worldwide=true, sorted by job count
```

Use the existing `verify-slugs.ts` infrastructure for slug generation and ATS verification. Extend it with new candidate sources.
