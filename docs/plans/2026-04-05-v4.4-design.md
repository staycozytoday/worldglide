# v4.4 Design: Filter Hardening + Company Expansion

Date: 2026-04-05

## Problem

Two structural filter gaps let false positives through:

1. **Bare `\bdesigner\b` catch-all** in `categorize.ts` matches any title containing "designer" — including hardware roles like "Avionics Designer" or "Battery Designer." The exclude list plays whack-a-mole but can't keep up.

2. **`isRemoteJob()` defaults to `true`** in `filter.ts` — if a job has no explicit office/hybrid signal, it passes. Jobs at "Long Beach, CA" with no remote keyword anywhere get treated as remote.

These violate the project's core principle: precision over recall.

## Design

### 1. Filter Hardening

#### A. Replace bare `\bdesigner\b` with qualified tech-creative patterns

Remove the catch-all `\bdesigner\b` (line 189 of categorize.ts). Replace with explicit qualified patterns that only match tech/digital creative roles:

```
product designer, ux designer, ui designer, visual designer,
graphic designer, web designer, brand designer, motion designer,
game designer, sound designer, experience designer, interaction designer,
instructional designer, content designer, service designer,
conversational designer, voice designer, accessibility designer,
design engineer, creative engineer, creative technologist
```

Add AI-native qualifier — any `ai + [creative role]` passes:
```
ai designer, ai product designer, ai native designer,
ai graphic designer, ai ux designer
```

General pattern: `/\b(ai\s+(?:native\s+)?)(product|ux|ui|visual|graphic|web|brand|motion)\s+designer\b/i`

#### B. Remove non-tech creative patterns

Drop from CREATIVE_PATTERNS:
- `industrial designer` — manufacturing, not tech
- `interior designer` — architecture, not tech

Keep:
- `game designer`, `sound designer` — tech-adjacent
- `instructional designer` — learning/ed-tech

#### C. Expand hardware exclude patterns

Add to EXCLUDE_PATTERNS:
```
/\b(avionics|battery|energy\s+storage|thermal|optical)\s+design/i
/\bhardware\s+(design\s+)?(manager|director|lead|head)/i
/\b(aerospace|defense|naval)\s+design/i
/\b(power\s+systems?|antenna|radar)\s+design/i
```

#### D. Tighten `isRemoteJob()` default

Current: defaults `true` when no signal found.
New: if location matches a US city+state pattern (e.g. "Long Beach, CA") or known office-only format, and no remote keyword exists anywhere in title/location/description, default to `false`.

Heuristic: `/\b[A-Z][a-z]+(?:\s[A-Z][a-z]+)*,\s*[A-Z]{2}\b/` (city, ST format) with no "remote" anywhere → reject.

### 2. Company Expansion

#### A. Add Japanese companies (6 verified)
Mercari, LINE, Recruit, Cybozu, Rakuten, SoftBank — all SmartRecruiters ATS, slugs verified live.

#### B. Prune dead 404 slugs
Remove 27 companies returning 404 errors (Solana Foundation, Grammarly, Doist, Farcaster, Replicate, etc.).

#### C. Mass discovery
Use existing discovery pipeline to scrape Greenhouse, Lever, and Ashby directories for tech companies with creative roles not yet in the company list.

Target: double from ~1,300 to 2,500+ companies.

## Success criteria

- Zero hardware/aerospace/energy "design" roles pass the creative filter
- Jobs with city+state locations and no remote signals are rejected
- Company count increases significantly
- Total live jobs on board increases (more companies, tighter filter = net positive)

## Files affected

- `src/lib/categorize.ts` — CREATIVE_PATTERNS, EXCLUDE_PATTERNS
- `src/lib/filter.ts` — `isRemoteJob()` default logic
- `src/lib/companies.ts` — add Japanese companies, prune dead slugs, add discovered companies
