## Task 3: Tighten `isRemoteJob()` default behavior

**Files:**
- Modify: `src/lib/filter.ts:782-804` (isRemoteJob function)

**Step 1: Add city+state detection**

Replace the current default-true return (line 802-803) with a city+state check:

```typescript
export function isRemoteJob(job: FilterableJob, fullDescription?: string): boolean {
  const loc = (job.location || "").toLowerCase().trim();

  // If location explicitly says office/hybrid → not remote
  for (const p of OFFICE_ONLY_PATTERNS) {
    if (p.test(loc)) return false;
  }

  // If location has a remote keyword → remote
  for (const kw of REMOTE_LOCATION_KEYWORDS) {
    if (loc.includes(kw)) return true;
  }

  // Location is a city/region name — check description for office signals
  const desc = (fullDescription || job.description || "").toLowerCase();
  for (const p of OFFICE_DESCRIPTION_SIGNALS) {
    if (p.test(desc)) return false;
  }

  // NEW: If location looks like "City, ST" (US) or "City, Country" with no
  // remote signal anywhere in title/location/description → reject.
  // Many companies list HQ city for on-site roles with no explicit "on-site" tag.
  const hasRemoteSignalAnywhere = REMOTE_LOCATION_KEYWORDS.some(kw =>
    loc.includes(kw) || desc.includes(kw)
  );

  if (!hasRemoteSignalAnywhere) {
    // US city+state: "Long Beach, CA", "San Francisco, California"
    const usCityState = /\b[a-z]+(?:\s[a-z]+)*,\s*[a-z]{2}\b/.test(loc)
      || /\b[a-z]+(?:\s[a-z]+)*,\s*[a-z]{4,}\b/.test(loc);
    // International city+country: "London, United Kingdom", "Berlin, Germany"
    const cityCountry = /\b[a-z]+(?:\s[a-z]+)*,\s*[a-z]+(?:\s[a-z]+)*$/.test(loc);

    if (usCityState || cityCountry) return false;
  }

  // No disqualifying signal found → treat as remote-eligible
  return true;
}
```

**Step 2: Validate against known job locations**

These should now FAIL (correctly rejected):

- "McCarran, NV" (Redwood Materials) → false

These should still PASS:

- "Remote - US" → matches remote keyword

- "" (empty) → no city+state pattern → defaults true
